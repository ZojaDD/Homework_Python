Сделать файл отслеживаемым:
git add <название файла>



Чтобы сохранить коммит в VSCode, нужно выполнить несколько шагов:


1. Перейди в раздел "Source Control" на боковой панели.

2. Введи сообщение для коммита в поле "Message".

3. Нажми на иконку галочки или используй команду Commit в меню, чтобы зафиксировать изменения.


Если все прошло успешно, изменения будут сохранены локально. Не забудь потом отправить их на удаленный репозиторий командой git push. (например: <git push origin lesson1>)



Чтобы создать pull request, следуй этим шагам:


1. Перейди на вкладку "Запросы на слияние" (Pull Requests) в твоем репозитории.

2. Нажми на кнопку "New Pull Request" (Новый запрос на слияние).

3. Выбери ветку lesson1 как исходную, и master (или main) как ветку, куда будет сливаться код.

4. Нажми "Create Pull Request" (Создать запрос на слияние).

5. Введи название и описание, если нужно, и снова нажми "Create Pull Request".

После этого ты сможешь скинуть ссылку на созданный pull request


После того как твой пул-реквест будет принят, ты можешь слить ветку lesson1 в master (main) следующим образом:

1. Перейди в репозиторий на GitHub.

2. Там ты увидишь кнопку "Merge pull request" в разделе твоего пул-реквеста.

3. Нажми на эту кнопку, чтобы слить изменения из ветки lesson1 в master (main).


Подтянуть изменения из удаленного реппозитория в локальный:
   git pull
